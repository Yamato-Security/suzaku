# Внесок та підтримка

## Внесок

Ми будемо раді будь-якій формі внеску.
Pull-запити, створення правил та зразки логів є найкращими, але запити на нові функції, повідомлення про помилки тощо також дуже вітаються.

Принаймні, **якщо вам подобаються наші інструменти та ресурси, будь ласка, поставте нам зірочку на GitHub та проявіть свою підтримку!**

## Надсилання повідомлень про помилки

* Будь ласка, надсилайте будь-які знайдені помилки [сюди.](https://github.com/Yamato-Security/suzaku/issues/new?assignees=&labels=bug&template=bug_report.md&title=%5Bbug%5D)
* Цей проєкт наразі активно підтримується, і ми будемо раді виправити будь-які повідомлені помилки.
* Якщо ви знайдете будь-які проблеми (хибні спрацювання, помилки тощо) у правилах Suzaku, будь ласка, повідомте про них на сторінці GitHub issues проєкту suzaku-rules [тут](https://github.com/Yamato-Security/suzaku-rules/issues/new).
* Якщо ви знайдете будь-які проблеми (хибні спрацювання, помилки тощо) у правилах Sigma, будь ласка, повідомте про них на сторінці GitHub issues вихідного проєкту SigmaHQ [тут](https://github.com/SigmaHQ/sigma/issues).

## Ліцензія

* Suzaku випускається під ліцензією [AGPLv3](https://www.gnu.org/licenses/agpl-3.0.en.html), а всі правила випускаються під ліцензією [Detection Rule License (DRL) 1.1](https://github.com/SigmaHQ/sigma/blob/master/LICENSE.Detection.Rules.md).
* Ви можете вільно використовувати Suzaku всередині організації, у SaaS-рішеннях, для консалтингової роботи тощо.
Однак, якщо ви вносите покращення, ми просимо вас зробити ці покращення з відкритим вихідним кодом та повернути їх проєкту.
