# Mitwirken & Support

## Mitwirken

Wir würden uns über jede Form von Beitrag freuen.
Pull Requests, das Erstellen von Regeln und Beispiel-Logs sind am besten, aber Funktionswünsche, das Melden von Fehlern usw. sind ebenfalls sehr willkommen.

Zumindest aber, **wenn Ihnen unsere Tools und Ressourcen gefallen, geben Sie uns bitte einen Stern auf GitHub und zeigen Sie Ihre Unterstützung!**

## Fehlermeldung

* Bitte melden Sie alle gefundenen Fehler [hier.](https://github.com/Yamato-Security/suzaku/issues/new?assignees=&labels=bug&template=bug_report.md&title=%5Bbug%5D)
* Dieses Projekt wird derzeit aktiv gepflegt und wir freuen uns, alle gemeldeten Fehler zu beheben.
* Wenn Sie Probleme (False Positives, Fehler usw.) mit Suzaku-Regeln finden, melden Sie diese bitte auf der suzaku-rules GitHub Issues-Seite [hier](https://github.com/Yamato-Security/suzaku-rules/issues/new).
* Wenn Sie Probleme (False Positives, Fehler usw.) mit Sigma-Regeln finden, melden Sie diese bitte auf der vorgelagerten SigmaHQ GitHub Issues-Seite [hier](https://github.com/SigmaHQ/sigma/issues).

## Lizenz

* Suzaku wird unter [AGPLv3](https://www.gnu.org/licenses/agpl-3.0.en.html) veröffentlicht und alle Regeln werden unter der [Detection Rule License (DRL) 1.1](https://github.com/SigmaHQ/sigma/blob/master/LICENSE.Detection.Rules.md) veröffentlicht.
* Sie dürfen Suzaku intern, für SaaS-Lösungen, für Beratungstätigkeiten usw. frei verwenden.
Wenn Sie jedoch Verbesserungen daran vornehmen, bitten wir Sie, diese Verbesserungen als Open Source bereitzustellen und an das Projekt zurückzugeben.
