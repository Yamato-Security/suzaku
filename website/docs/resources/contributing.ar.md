# المساهمة والدعم

## المساهمة

نرحب بأي شكل من أشكال المساهمة.
طلبات السحب (Pull requests) وإنشاء القواعد وعينات السجلات هي الأفضل، ولكن طلبات الميزات والإبلاغ عن الأخطاء وما إلى ذلك مرحب بها أيضاً.

على الأقل، **إذا أعجبتك أدواتنا ومواردنا، فالرجاء منحنا نجمة على GitHub وإظهار دعمك!**

## الإبلاغ عن الأخطاء

* الرجاء الإبلاغ عن أي أخطاء تجدها [هنا.](https://github.com/Yamato-Security/suzaku/issues/new?assignees=&labels=bug&template=bug_report.md&title=%5Bbug%5D)
* تتم صيانة هذا المشروع بنشاط حالياً ويسعدنا إصلاح أي أخطاء يتم الإبلاغ عنها.
* إذا وجدت أي مشكلات (نتائج إيجابية خاطئة، أخطاء، إلخ...) مع قواعد Suzaku، فالرجاء الإبلاغ عنها إلى صفحة مشكلات suzaku-rules على GitHub [هنا](https://github.com/Yamato-Security/suzaku-rules/issues/new).
* إذا وجدت أي مشكلات (نتائج إيجابية خاطئة، أخطاء، إلخ...) مع قواعد Sigma، فالرجاء الإبلاغ عنها إلى صفحة مشكلات SigmaHQ الأصلية (upstream) على GitHub [هنا](https://github.com/SigmaHQ/sigma/issues).

## الترخيص

* يتم إصدار Suzaku بموجب [AGPLv3](https://www.gnu.org/licenses/agpl-3.0.en.html) ويتم إصدار جميع القواعد بموجب [Detection Rule License (DRL) 1.1](https://github.com/SigmaHQ/sigma/blob/master/LICENSE.Detection.Rules.md).
* يمكنك استخدام Suzaku بحرية داخلياً، وفي حلول SaaS، ولأعمال الاستشارات، إلخ...
ومع ذلك، إذا أجريت تحسينات عليه، فإننا نطلب منك جعل تلك التحسينات مفتوحة المصدر والمساهمة بها في المشروع.
