# การมีส่วนร่วมและการสนับสนุน

## การมีส่วนร่วม

เรายินดีรับการมีส่วนร่วมในทุกรูปแบบ
Pull request การสร้างกฎ และตัวอย่างล็อกเป็นสิ่งที่ดีที่สุด แต่คำขอฟีเจอร์ การแจ้งบั๊กให้เราทราบ และอื่น ๆ ก็ยินดีรับเช่นกัน

อย่างน้อยที่สุด **หากคุณชื่นชอบเครื่องมือและทรัพยากรของเรา โปรดให้ดาวเราบน GitHub และแสดงการสนับสนุนของคุณ!**

## การส่งรายงานบั๊ก

* โปรดส่งบั๊กที่คุณพบ [ที่นี่](https://github.com/Yamato-Security/suzaku/issues/new?assignees=&labels=bug&template=bug_report.md&title=%5Bbug%5D)
* โปรเจกต์นี้กำลังได้รับการดูแลอย่างต่อเนื่องในปัจจุบัน และเรายินดีที่จะแก้ไขบั๊กใด ๆ ที่ได้รับการรายงาน
* หากคุณพบปัญหาใด ๆ (false positive, บั๊ก ฯลฯ) กับกฎของ Suzaku โปรดรายงานไปยังหน้า GitHub issues ของ suzaku-rules [ที่นี่](https://github.com/Yamato-Security/suzaku-rules/issues/new)
* หากคุณพบปัญหาใด ๆ (false positive, บั๊ก ฯลฯ) กับกฎของ Sigma โปรดรายงานไปยังหน้า GitHub issues ต้นทางของ SigmaHQ [ที่นี่](https://github.com/SigmaHQ/sigma/issues)

## ใบอนุญาต

* Suzaku เผยแพร่ภายใต้ [AGPLv3](https://www.gnu.org/licenses/agpl-3.0.en.html) และกฎทั้งหมดเผยแพร่ภายใต้ [Detection Rule License (DRL) 1.1](https://github.com/SigmaHQ/sigma/blob/master/LICENSE.Detection.Rules.md)
* คุณสามารถใช้ Suzaku ได้อย่างอิสระภายในองค์กร โซลูชัน SaaS งานที่ปรึกษา ฯลฯ
อย่างไรก็ตาม หากคุณปรับปรุงมัน เราขอให้คุณเปิดเผยซอร์สโค้ดของการปรับปรุงเหล่านั้นและตอบแทนกลับสู่โปรเจกต์
