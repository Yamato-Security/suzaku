# Contribuição e Suporte

## Contribuição

Adoraríamos receber qualquer forma de contribuição.
Pull requests, criação de regras e logs de exemplo são as melhores, mas solicitações de funcionalidades, notificações sobre bugs, etc... também são muito bem-vindas.

No mínimo, **se você gosta das nossas ferramentas e recursos, então por favor nos dê uma estrela no GitHub e demonstre seu apoio!**

## Envio de Bugs

* Por favor, envie quaisquer bugs que você encontrar [aqui.](https://github.com/Yamato-Security/suzaku/issues/new?assignees=&labels=bug&template=bug_report.md&title=%5Bbug%5D)
* Este projeto está sendo ativamente mantido no momento e ficamos felizes em corrigir quaisquer bugs reportados.
* Se você encontrar qualquer problema (falsos positivos, bugs, etc...) com as regras do Suzaku, por favor reporte-os na página de issues do GitHub do suzaku-rules [aqui](https://github.com/Yamato-Security/suzaku-rules/issues/new).
* Se você encontrar qualquer problema (falsos positivos, bugs, etc...) com as regras do Sigma, por favor reporte-os na página de issues do GitHub do SigmaHQ upstream [aqui](https://github.com/SigmaHQ/sigma/issues).

## Licença

* O Suzaku é distribuído sob a [AGPLv3](https://www.gnu.org/licenses/agpl-3.0.en.html) e todas as regras são distribuídas sob a [Detection Rule License (DRL) 1.1](https://github.com/SigmaHQ/sigma/blob/master/LICENSE.Detection.Rules.md).
* Você pode usar livremente o Suzaku internamente, em soluções SaaS, para trabalhos de consultoria, etc...
No entanto, se você fizer melhorias nele, pedimos que torne essas melhorias open-source e contribua de volta ao projeto.
