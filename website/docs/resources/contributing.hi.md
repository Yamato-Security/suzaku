# योगदान और समर्थन

## योगदान

हम किसी भी प्रकार के योगदान का स्वागत करते हैं।
Pull request, नियम निर्माण और सैंपल लॉग सबसे अच्छे हैं, लेकिन फीचर अनुरोध, हमें बग्स के बारे में सूचित करना, आदि भी बहुत स्वागत योग्य हैं।

कम से कम, **यदि आपको हमारे टूल और संसाधन पसंद आते हैं तो कृपया हमें GitHub पर एक स्टार दें और अपना समर्थन दिखाएँ!**

## बग सबमिशन

* कृपया आपको मिलने वाले किसी भी बग को [यहाँ](https://github.com/Yamato-Security/suzaku/issues/new?assignees=&labels=bug&template=bug_report.md&title=%5Bbug%5D) सबमिट करें।
* यह प्रोजेक्ट वर्तमान में सक्रिय रूप से अनुरक्षित है और हम रिपोर्ट किए गए किसी भी बग को ठीक करने में खुश हैं।
* यदि आपको Suzaku नियमों में कोई समस्या (false positive, बग, आदि) मिलती है, तो कृपया उन्हें suzaku-rules GitHub issues पृष्ठ पर [यहाँ](https://github.com/Yamato-Security/suzaku-rules/issues/new) रिपोर्ट करें।
* यदि आपको Sigma नियमों में कोई समस्या (false positive, बग, आदि) मिलती है, तो कृपया उन्हें अपस्ट्रीम SigmaHQ GitHub issues पृष्ठ पर [यहाँ](https://github.com/SigmaHQ/sigma/issues) रिपोर्ट करें।

## लाइसेंस

* Suzaku को [AGPLv3](https://www.gnu.org/licenses/agpl-3.0.en.html) के तहत जारी किया गया है और सभी नियम [Detection Rule License (DRL) 1.1](https://github.com/SigmaHQ/sigma/blob/master/LICENSE.Detection.Rules.md) के तहत जारी किए गए हैं।
* आप Suzaku का स्वतंत्र रूप से आंतरिक रूप से, SaaS समाधानों में, परामर्श कार्य के लिए, आदि उपयोग कर सकते हैं।
हालाँकि, यदि आप इसमें सुधार करते हैं, तो हम आपसे अनुरोध करते हैं कि उन सुधारों को ओपन-सोर्स करें और प्रोजेक्ट को वापस दें।
