# ပံ့ပိုးကူညီခြင်းနှင့် ပံ့ပိုးမှု

## ပံ့ပိုးကူညီခြင်း

ကျွန်ုပ်တို့သည် မည်သည့်ပုံစံဖြင့်မဆို ပံ့ပိုးကူညီမှုကို ကြိုဆိုပါသည်။
Pull request များ၊ rule ဖန်တီးခြင်းနှင့် နမူနာ log များသည် အကောင်းဆုံးဖြစ်သော်လည်း feature တောင်းဆိုခြင်း၊ bug များကို ကျွန်ုပ်တို့အား အကြောင်းကြားခြင်း စသည်တို့ကိုလည်း အလွန်ကြိုဆိုပါသည်။

အနည်းဆုံးအားဖြင့်၊ **ကျွန်ုပ်တို့၏ tool များနှင့် resource များကို သင်နှစ်သက်ပါက ကျေးဇူးပြု၍ GitHub တွင် ကျွန်ုပ်တို့အား star ပေး၍ သင်၏ ပံ့ပိုးမှုကို ပြသပါ!**

## Bug တင်ပြခြင်း

* သင်တွေ့ရှိသော မည်သည့် bug ကိုမဆို ကျေးဇူးပြု၍ [ဤနေရာတွင်](https://github.com/Yamato-Security/suzaku/issues/new?assignees=&labels=bug&template=bug_report.md&title=%5Bbug%5D) တင်ပြပါ။
* ဤ project ကို လက်ရှိအချိန်တွင် တက်ကြွစွာ ထိန်းသိမ်းနေပြီး တင်ပြလာသော မည်သည့် bug ကိုမဆို ဖြေရှင်းပေးရန် ကျွန်ုပ်တို့ ဝမ်းသာပါသည်။
* Suzaku rule များတွင် မည်သည့်ပြဿနာ (false positive များ၊ bug များ စသည်ဖြင့်) ကိုမဆို တွေ့ရှိပါက၊ ၎င်းတို့ကို suzaku-rules GitHub issues စာမျက်နှာ [ဤနေရာတွင်](https://github.com/Yamato-Security/suzaku-rules/issues/new) ကျေးဇူးပြု၍ တင်ပြပါ။
* Sigma rule များတွင် မည်သည့်ပြဿနာ (false positive များ၊ bug များ စသည်ဖြင့်) ကိုမဆို တွေ့ရှိပါက၊ ၎င်းတို့ကို upstream SigmaHQ GitHub issues စာမျက်နှာ [ဤနေရာတွင်](https://github.com/SigmaHQ/sigma/issues) ကျေးဇူးပြု၍ တင်ပြပါ။

## လိုင်စင်

* Suzaku ကို [AGPLv3](https://www.gnu.org/licenses/agpl-3.0.en.html) အောက်တွင် ထုတ်ဝေထားပြီး rule အားလုံးကို [Detection Rule License (DRL) 1.1](https://github.com/SigmaHQ/sigma/blob/master/LICENSE.Detection.Rules.md) အောက်တွင် ထုတ်ဝေထားသည်။
* သင်သည် Suzaku ကို အတွင်းပိုင်း၊ SaaS solution များ၊ consulting လုပ်ငန်းများ စသည်တို့အတွက် လွတ်လပ်စွာ အသုံးပြုနိုင်သည်။
သို့သော်လည်း၊ သင်သည် ၎င်းအား တိုးတက်ကောင်းမွန်အောင် ပြုလုပ်ပါက၊ ထိုတိုးတက်မှုများကို open-source ပြုလုပ်၍ project သို့ ပြန်လည်ပေးအပ်ရန် ကျွန်ုပ်တို့ တောင်းဆိုပါသည်။
