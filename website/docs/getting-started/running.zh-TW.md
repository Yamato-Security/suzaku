# 執行 Suzaku

## Windows

在命令提示字元/PowerShell 提示字元或 Windows Terminal 中，直接執行對應的 32 位元或 64 位元 Windows 二進位檔案即可。

### 掃描路徑中含有空格的檔案或目錄時發生錯誤

當您使用 Windows 內建的命令提示字元或 PowerShell 提示字元時，如果您的檔案或目錄路徑中含有空格，您可能會收到 Suzaku 無法載入任何檔案的錯誤訊息。
為了正確載入日誌檔案，請務必執行以下步驟：
1. 將檔案或目錄路徑用雙引號括起來。
2. 如果是目錄路徑，請確認最後一個字元不要包含反斜線。

### 字元無法正確顯示

在 Windows 上使用預設字型 `Lucida Console` 時，標誌與表格中使用的各種字元將無法正確顯示。
您應該將字型變更為 `Consalas` 來修正此問題。

## Linux

您首先需要讓二進位檔案可執行。

```bash
chmod +x ./suzaku
```

接著從 Suzaku 根目錄執行它：

```bash
./suzaku
```

## macOS

在終端機或 [iTerm2](https://iterm2.com/) 中，您首先需要讓二進位檔案可執行。

```bash
chmod +x ./suzaku
```

接著，嘗試從 Suzaku 根目錄執行它：

```bash
./suzaku
```

在最新版本的 macOS 上，當您嘗試執行它時，可能會收到安全性錯誤。
點選「取消」，然後從「系統偏好設定」開啟「安全性與隱私」，並在「一般」分頁中點選「仍要允許」。
之後，再次嘗試執行它。

```bash
./suzaku
```

此時會彈出一個警告，只要點選「開啟」即可。
現在您應該能夠執行 suzaku 了。
