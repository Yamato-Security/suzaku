# Запуск Suzaku

## Windows

У командному рядку / PowerShell або Windows Terminal просто запустіть відповідний 32-розрядний або 64-розрядний двійковий файл Windows.

### Помилка під час спроби сканувати файл або каталог з пробілом у шляху

Під час використання вбудованого командного рядка або PowerShell у Windows ви можете отримати помилку про те, що Suzaku не зміг завантажити жодного файлу, якщо у шляху до вашого файлу або каталогу є пробіл.
Щоб правильно завантажити файли журналів, обов'язково виконайте наступне:
1. Візьміть шлях до файлу або каталогу в подвійні лапки.
2. Якщо це шлях до каталогу, переконайтеся, що останнім символом не є зворотна коса риска.

### Символи відображаються некоректно

Зі стандартним шрифтом `Lucida Console` у Windows різні символи, що використовуються в логотипі та таблицях, відображатимуться некоректно.
Щоб виправити це, вам слід змінити шрифт на `Consalas`.

## Linux

Спершу вам потрібно зробити двійковий файл виконуваним.

```bash
chmod +x ./suzaku
```

Потім запустіть його з кореневого каталогу Suzaku:

```bash
./suzaku
```

## macOS

З Terminal або [iTerm2](https://iterm2.com/) спершу вам потрібно зробити двійковий файл виконуваним.

```bash
chmod +x ./suzaku
```

Потім спробуйте запустити його з кореневого каталогу Suzaku:

```bash
./suzaku
```

В останній версії macOS під час спроби запустити його ви можете отримати помилку безпеки.
Натисніть "Cancel", потім у System Preferences відкрийте "Security & Privacy" і на вкладці General натисніть "Allow Anyway".
Після цього спробуйте запустити його знову.

```bash
./suzaku
```

З'явиться попередження, тож просто натисніть "Open".
Тепер ви маєте змогу запустити suzaku.
