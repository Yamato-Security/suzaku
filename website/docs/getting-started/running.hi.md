# Suzaku चलाना

## Windows

Command/PowerShell Prompt या Windows Terminal में, बस उपयुक्त 32-bit या 64-bit Windows बाइनरी चलाएँ।

### पाथ में स्पेस वाली किसी फ़ाइल या डायरेक्टरी को स्कैन करने का प्रयास करते समय त्रुटि

Windows में बिल्ट-इन Command या PowerShell prompt का उपयोग करते समय, यदि आपकी फ़ाइल या डायरेक्टरी पाथ में स्पेस है तो आपको एक त्रुटि मिल सकती है कि Suzaku कोई भी फ़ाइल लोड करने में असमर्थ रहा।
लॉग फ़ाइलों को सही ढंग से लोड करने के लिए, निम्नलिखित अवश्य करें:
1. फ़ाइल या डायरेक्टरी पाथ को डबल कोट्स में संलग्न करें।
2. यदि यह एक डायरेक्टरी पाथ है, तो सुनिश्चित करें कि अंतिम वर्ण के रूप में बैकस्लैश शामिल न करें।

### वर्ण सही ढंग से प्रदर्शित नहीं हो रहे

Windows पर डिफ़ॉल्ट फ़ॉन्ट `Lucida Console` के साथ, लोगो और तालिकाओं में उपयोग किए गए विभिन्न वर्ण सही ढंग से प्रदर्शित नहीं होंगे।
इसे ठीक करने के लिए आपको फ़ॉन्ट को `Consalas` में बदलना चाहिए।

## Linux

आपको पहले बाइनरी को निष्पादन योग्य बनाना होगा।

```bash
chmod +x ./suzaku
```

फिर इसे Suzaku रूट डायरेक्टरी से चलाएँ:

```bash
./suzaku
```

## macOS

Terminal या [iTerm2](https://iterm2.com/) से, आपको पहले बाइनरी को निष्पादन योग्य बनाना होगा।

```bash
chmod +x ./suzaku
```

फिर, इसे Suzaku रूट डायरेक्टरी से चलाने का प्रयास करें:

```bash
./suzaku
```

macOS के नवीनतम संस्करण पर, जब आप इसे चलाने का प्रयास करते हैं तो आपको एक सुरक्षा त्रुटि मिल सकती है।
"Cancel" पर क्लिक करें और फिर System Preferences से, "Security & Privacy" खोलें और General टैब से, "Allow Anyway" पर क्लिक करें।
इसके बाद, इसे फिर से चलाने का प्रयास करें।

```bash
./suzaku
```

एक चेतावनी पॉप अप होगी इसलिए बस "Open" पर क्लिक करें।
अब आप suzaku चलाने में सक्षम होने चाहिए।
