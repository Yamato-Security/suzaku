# تشغيل Suzaku

## Windows

في موجّه أوامر Command/PowerShell أو في Windows Terminal، ما عليك سوى تشغيل ملف Windows الثنائي المناسب بنظام 32-بت أو 64-بت.

### خطأ عند محاولة فحص ملف أو دليل يحتوي مساره على مسافة

عند استخدام موجّه Command أو PowerShell المدمج في Windows، قد تتلقى خطأً بأن Suzaku لم يتمكن من تحميل أي ملفات إذا كانت هناك مسافة في مسار الملف أو الدليل.
لتحميل ملفات السجل بشكل صحيح، تأكد من القيام بما يلي:
1. ضع مسار الملف أو الدليل بين علامتي اقتباس مزدوجتين.
2. إذا كان مسار دليل، فتأكد من عدم تضمين شرطة مائلة عكسية كآخر حرف.

### عدم عرض الأحرف بشكل صحيح

مع الخط الافتراضي `Lucida Console` على Windows، لن يتم عرض مختلف الأحرف المستخدمة في الشعار والجداول بشكل صحيح.
يجب عليك تغيير الخط إلى `Consalas` لإصلاح هذه المشكلة.

## Linux

تحتاج أولاً إلى جعل الملف الثنائي قابلاً للتنفيذ.

```bash
chmod +x ./suzaku
```

ثم قم بتشغيله من الدليل الجذر لـ Suzaku:

```bash
./suzaku
```

## macOS

من Terminal أو [iTerm2](https://iterm2.com/)، تحتاج أولاً إلى جعل الملف الثنائي قابلاً للتنفيذ.

```bash
chmod +x ./suzaku
```

ثم، حاول تشغيله من الدليل الجذر لـ Suzaku:

```bash
./suzaku
```

في أحدث إصدار من macOS، قد تتلقى خطأ أمان عند محاولة تشغيله.
انقر على "Cancel" ثم من System Preferences، افتح "Security & Privacy" ومن علامة التبويب General، انقر على "Allow Anyway".
بعد ذلك، حاول تشغيله مرة أخرى.

```bash
./suzaku
```

سيظهر تحذير منبثق فما عليك سوى النقر على "Open".
يجب أن تكون الآن قادراً على تشغيل suzaku.
