# Suzaku 실행하기

## Windows

명령/PowerShell 프롬프트 또는 Windows Terminal에서 적절한 32비트 또는 64비트 Windows 바이너리를 실행하기만 하면 됩니다.

### 경로에 공백이 있는 파일이나 디렉터리를 스캔할 때 발생하는 오류

Windows의 기본 제공 명령 또는 PowerShell 프롬프트를 사용할 때, 파일이나 디렉터리 경로에 공백이 있으면 Suzaku가 파일을 로드하지 못했다는 오류가 발생할 수 있습니다.
로그 파일을 올바르게 로드하려면 다음을 반드시 수행하십시오:
1. 파일 또는 디렉터리 경로를 큰따옴표로 묶으십시오.
2. 디렉터리 경로인 경우, 마지막 문자에 백슬래시를 포함하지 않도록 하십시오.

### 문자가 올바르게 표시되지 않음

Windows의 기본 글꼴인 `Lucida Console`에서는 로고와 테이블에 사용되는 다양한 문자가 제대로 표시되지 않습니다.
이를 해결하려면 글꼴을 `Consalas`로 변경해야 합니다.

## Linux

먼저 바이너리를 실행 가능하게 만들어야 합니다.

```bash
chmod +x ./suzaku
```

그런 다음 Suzaku 루트 디렉터리에서 실행하십시오:

```bash
./suzaku
```

## macOS

터미널 또는 [iTerm2](https://iterm2.com/)에서 먼저 바이너리를 실행 가능하게 만들어야 합니다.

```bash
chmod +x ./suzaku
```

그런 다음 Suzaku 루트 디렉터리에서 실행해 보십시오:

```bash
./suzaku
```

최신 버전의 macOS에서는 실행하려고 할 때 보안 오류가 발생할 수 있습니다.
"취소"를 클릭한 다음, 시스템 환경설정에서 "보안 및 개인 정보 보호"를 열고 일반 탭에서 "그래도 허용"을 클릭하십시오.
그 후 다시 실행해 보십시오.

```bash
./suzaku
```

경고가 표시되면 "열기"를 클릭하기만 하면 됩니다.
이제 suzaku를 실행할 수 있을 것입니다.
