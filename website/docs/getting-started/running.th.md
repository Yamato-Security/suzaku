# การรัน Suzaku

## Windows

ใน Command/PowerShell Prompt หรือ Windows Terminal เพียงแค่รันไบนารี Windows แบบ 32-bit หรือ 64-bit ที่เหมาะสม

### ข้อผิดพลาดเมื่อพยายามสแกนไฟล์หรือไดเรกทอรีที่มีช่องว่างในพาธ

เมื่อใช้ Command หรือ PowerShell prompt ที่มาพร้อมกับ Windows คุณอาจได้รับข้อผิดพลาดว่า Suzaku ไม่สามารถโหลดไฟล์ใด ๆ ได้ หากมีช่องว่างในพาธของไฟล์หรือไดเรกทอรีของคุณ
เพื่อให้โหลดไฟล์ล็อกได้อย่างถูกต้อง โปรดทำตามขั้นตอนต่อไปนี้:
1. คร่อมพาธของไฟล์หรือไดเรกทอรีด้วยเครื่องหมายอัญประกาศคู่
2. หากเป็นพาธของไดเรกทอรี ตรวจสอบให้แน่ใจว่าคุณไม่ได้ใส่เครื่องหมายแบ็กสแลชเป็นอักขระตัวสุดท้าย

### อักขระแสดงผลไม่ถูกต้อง

ด้วยฟอนต์เริ่มต้น `Lucida Console` บน Windows อักขระต่าง ๆ ที่ใช้ในโลโก้และตารางจะแสดงผลไม่ถูกต้อง
คุณควรเปลี่ยนฟอนต์เป็น `Consalas` เพื่อแก้ไขปัญหานี้

## Linux

ก่อนอื่นคุณต้องทำให้ไบนารีสามารถรันได้

```bash
chmod +x ./suzaku
```

จากนั้นรันจากไดเรกทอรีราก (root) ของ Suzaku:

```bash
./suzaku
```

## macOS

จาก Terminal หรือ [iTerm2](https://iterm2.com/) ก่อนอื่นคุณต้องทำให้ไบนารีสามารถรันได้

```bash
chmod +x ./suzaku
```

จากนั้นลองรันจากไดเรกทอรีราก (root) ของ Suzaku:

```bash
./suzaku
```

บน macOS เวอร์ชันล่าสุด คุณอาจได้รับข้อผิดพลาดด้านความปลอดภัยเมื่อพยายามรัน
คลิก "Cancel" จากนั้นไปที่ System Preferences เปิด "Security & Privacy" และจากแท็บ General คลิก "Allow Anyway"
หลังจากนั้นลองรันอีกครั้ง

```bash
./suzaku
```

จะมีคำเตือนปรากฏขึ้น เพียงคลิก "Open"
ตอนนี้คุณควรจะสามารถรัน suzaku ได้แล้ว
