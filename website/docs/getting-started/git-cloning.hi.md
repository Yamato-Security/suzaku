# Git Cloning

आप निम्नलिखित कमांड के साथ रिपॉज़िटरी को `git clone` कर सकते हैं और सोर्स कोड से बाइनरी कंपाइल कर सकते हैं:

**Warning:** रिपॉज़िटरी की मुख्य ब्रांच विकास उद्देश्यों के लिए है इसलिए आप ऐसी नई सुविधाओं तक पहुँच सकते हैं जो अभी तक आधिकारिक रूप से रिलीज़ नहीं हुई हैं, हालाँकि, इसमें बग्स हो सकते हैं इसलिए इसे अस्थिर मानें।

```bash
git clone https://github.com/Yamato-Security/suzaku.git --recursive
```

> **Note:** यदि आप `--recursive` विकल्प का उपयोग करना भूल जाते हैं, तो `rules` फ़ोल्डर, जो एक git submodule के रूप में प्रबंधित किया जाता है, क्लोन नहीं होगा।

आप `git pull --recurse-submodules` के साथ `rules` फ़ोल्डर को सिंक कर सकते हैं और नवीनतम Suzaku नियम प्राप्त कर सकते हैं या निम्नलिखित कमांड का उपयोग कर सकते हैं:

```bash
./suzaku update-rules
```

यदि अपडेट विफल हो जाता है, तो आपको `rules` फ़ोल्डर का नाम बदलना होगा और फिर से प्रयास करना होगा।

>> Caution: अपडेट करते समय, `rules` फ़ोल्डर में नियम और कॉन्फ़िग फ़ाइलें [suzaku-rules](https://github.com/Yamato-Security/suzaku-rules) रिपॉज़िटरी में नवीनतम नियमों और कॉन्फ़िग फ़ाइलों से बदल दी जाती हैं।
>> मौजूदा फ़ाइलों में आपके द्वारा किए गए कोई भी परिवर्तन अधिलेखित हो जाएँगे, इसलिए हम अनुशंसा करते हैं कि अपडेट करने से पहले आप संपादित की गई किसी भी फ़ाइल का बैकअप बना लें।
>> यदि आप `rules` फ़ोल्डर के अंदर **नए** नियम जोड़ते हैं, तो अपडेट करते समय वे **अधिलेखित** या हटाए **नहीं** जाएँगे।
