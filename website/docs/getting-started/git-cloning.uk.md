# Клонування через Git

Ви можете виконати `git clone` репозиторію за допомогою наведеної нижче команди та скомпілювати бінарний файл із вихідного коду:

**Попередження:** Гілка main репозиторію призначена для розробки, тому ви можете отримати доступ до нових функцій, які ще офіційно не випущені, однак можуть бути помилки, тому вважайте її нестабільною.

```bash
git clone https://github.com/Yamato-Security/suzaku.git --recursive
```

> **Примітка:** Якщо ви забудете використати опцію `--recursive`, папка `rules`, яка керується як підмодуль git, не буде клонована.

Ви можете синхронізувати папку `rules` та отримати найновіші правила Suzaku за допомогою `git pull --recurse-submodules` або скористатися наведеною нижче командою:

```bash
./suzaku update-rules
```

Якщо оновлення не вдалося, можливо, вам потрібно перейменувати папку `rules` і спробувати знову.

>> Увага: Під час оновлення правила та файли конфігурації в папці `rules` замінюються найновішими правилами та файлами конфігурації з репозиторію [suzaku-rules](https://github.com/Yamato-Security/suzaku-rules).
>> Будь-які зміни, які ви вносите до наявних файлів, будуть перезаписані, тому ми рекомендуємо створювати резервні копії будь-яких файлів, які ви редагуєте, перед оновленням.
>> Якщо ви додасте **нові** правила всередині папки `rules`, вони **не** будуть перезаписані або видалені під час оновлення.
